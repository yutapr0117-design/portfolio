# package-lock.json

## What

npm の lockfile (lockfileVersion 3)。`devDependencies` の transitive 含めた pin 完全 graph。`npm ci` が読んで reproducible install を実現。

## Why

reproducible builds が CI の前提。lockfile が無いと毎回 transitive dep が変動して flaky test の原因に。

## How (usage)

```
npm ci
  └─ package-lock.json を読む
  └─ exact versions を install
  └─ node_modules/ を構築
```

## Constraints

- **Check 23**: JSON 構文 valid
- **Check 38**: package.json と sync (name / version / lockfileVersion 3)
- **手動編集禁止**: `npm install` 経由でのみ更新

## Change impact

- 手動編集 → npm ci 失敗 / install 不整合
- 削除 → reproducible build 失う
