# SECURITY.md

## What

セキュリティ脆弱性報告ポリシー。GitHub Security Advisory の標準形式。

## Why

GitHub が SECURITY.md の存在を見て「Security policy」リンクを表示。第三者が脆弱性を発見したときの正規 channel を明示。

## How (usage)

GitHub UI に Security tab → SECURITY.md 内容表示。第三者は記載された channel で報告。

## Constraints

- GitHub の慣例 (root SECURITY.md)

## Change impact

- 報告 channel 変更 → 公開連絡先の整合

## Audience-specific notes

### For AI agents
- 役割タグ: `security-policy`, `vulnerability-reporting`

### For human engineers (新卒レベル)
- 脆弱性を発見したらここの channel で報告

### For third parties (監査 / 採用 / 研究)
- 健全な OSS practice の最低限実装
