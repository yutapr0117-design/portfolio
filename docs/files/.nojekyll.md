# .nojekyll

## What

GitHub Pages の Jekyll 処理を **無効化** する空ファイル。中身ゼロ。

## Why

GitHub Pages はデフォルトで Jekyll を実行する。Jekyll は underscore prefix のファイル (`_template.md` 等) を「draft」として除外したり、Markdown を HTML 変換したり、`.txt` の Content-Type を意図せず変えたりする可能性。

`.nojekyll` を置くと Jekyll 処理を完全 skip して、ファイルを **raw のまま配信** する。AIO 戦略上、`llms.txt` / `llms-full.txt` の Content-Type / 内容が変質しないことが重要。

## How (usage)

GitHub Pages が deploy 時に `.nojekyll` の存在を確認 → Jekyll bypass → raw 配信。

## Constraints

- **Check 通り**: architecture-validation step 7 が `.nojekyll` 不在を WARNING として検出
- AIO 戦略上 (削除すると llms.txt が誤変換される可能性)

## Change impact

- 削除 → Jekyll が動き、`_*` prefix ファイルが draft 扱い等で配信されなくなる + `.txt` Content-Type 変質
