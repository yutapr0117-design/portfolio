# .stylelintrc.json

## What

Stylelint 設定 JSON。CSS の specificity / 変数整合 / lint rule を定義。`severity: warning` で既存コードへのヒットを CI ブロックしないルールも含む。

## Why

外部 CSS フレームワークなしの単一 `style.css` (2,156 行) の品質保護。stylelint プラグインは現状 `plugins: []` (declaration-strict-value プラグインは過去使用されたが削除)。

## How (usage)

```
npm run lint:css
  └─ python3 .github/scripts/check_css_stylelint.py
       └─ npx stylelint style.css (.stylelintrc.json を読む)
```

## Constraints

- **Check 23**: JSON 構文 valid

## Change impact

- rule 追加 → style.css 既存記述への影響大 (warning か error か慎重判断)
