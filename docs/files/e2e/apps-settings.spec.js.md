# e2e/apps-settings.spec.js

## What

Settings アプリの e2e。フルバックアップ export・部分 export・手動プロジェクト追加と Projects 反映・同名プロジェクトの unique slug・hide/unhide・reorder・削除 (confirm 受諾/キャンセル)・reset (confirm 受諾/キャンセル)・normalize without data loss・snapshot 保存/clear・AI history length bound (#230)・非配列 history/project 耐性 (#93)・JSON import upsert round-trip・snapshot restore を検証する。

## Why

データ管理面 (export/import/snapshot/CRUD) の data-fidelity と外部 ingestion 正規化を機能ゲートで守る。settings upsert data-loss (#192) 等の回帰を封じる。

## How (usage)

```
npm run test:e2e
  └─ Playwright + http-server (playwright.config.cjs testDir: ./e2e)
  └─ e2e/*.spec.js は testMatch デフォルト (**/*.spec.js) で自動 discovery
  └─ behavior assertions (screenshot は portfolio.spec.js に残置)
```
